# Glossary

**Session**: A server-side record associating an authenticated user with a set of issued tokens. Sessions have a fixed inactivity timeout and are invalidated on logout or expiry.

**Token**: A signed, self-contained credential issued to a client after successful authentication. Tokens are opaque to clients and must be validated by the Auth Service or a trusted introspection endpoint.

**MFA (Multi-Factor Authentication)**: An authentication mechanism requiring the user to present two or more independent factors: something they know (password), something they have (OTP device), or something they are (biometric).

**OTP (One-Time Password)**: A single-use numeric code valid for a short time window, delivered via SMS or generated by an authenticator app.

**TOTP (Time-Based OTP)**: An OTP variant derived from a shared secret and the current Unix timestamp, standardised in RFC 6238.

**Refresh Token**: A long-lived credential used to obtain new access tokens without requiring the user to re-authenticate. Rotated on each use.

**RBAC (Role-Based Access Control)**: An authorisation model in which permissions are grouped into roles, and roles are assigned to users rather than granting permissions directly.
